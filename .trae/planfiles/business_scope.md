# 业务边界（Scope，硬约束）

## 必须实现（Must-Have）

1. 进销存完整链路  
   - 入库需支持批次与保质期  
   - 销售数据作为算法唯一数据源  
   - 库存扣减需可追溯  

2. 智能补货闭环  
   - 基于历史销量的预测计算（时间序列分析）  
   - 自动生成补货建议  
   - 支持人工确认后生效  

3. 预警机制  
   - 库存安全阈值预警  
   - 商品临期 / 过期预警  

4. 可视化展示  
   - 首页仪表盘  
   - 至少包含：销量趋势、预测结果、待处理事项  

5. 多角色协同  
   - 至少包含：店长 / 库管 / 收银 / 系统管理员 / 顾客  
   - 角色分工必须形成业务闭环，而非仅权限区分  

## 明确禁止（Out of Scope）

- 真实支付网关（仅模拟支付成功）  
- 多门店 / 多租户架构  
- 真实硬件设备对接  
- 深度学习模型（如 LSTM / Transformer）
