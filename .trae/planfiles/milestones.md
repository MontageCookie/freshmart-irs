# 全周期核心交付物（Milestones）

## 工程交付物（必须真实存在）

- 数据库 DDL（支持批次、库存、销售、补货）
- 后端 API 接口定义文档
- 核心业务逻辑实现代码（尤其是库存扣减与补货计算）
- 算法实现代码（Python / Java / 混合均可）
- 可运行的完整系统

## 论文 / 答辩交付物（必须可截图）

1. 数据库 E-R 图  
   - 明确展示商品、销售记录、库存批次之间关系  
2. 核心业务流程图  
   - 入库 → 销售 → 扣减 → 补货 → 再入库  
3. 算法流程图  
   - 销量数据 → 时间序列分析 → 补货建议生成  
4. 算法效果对比图  
   - 历史真实销量 vs 预测销量  
5. 系统功能截图  
   - 首页仪表盘  
   - 补货建议页面  
   - 库存批次管理页面  

任一阶段完成时，对应交付物必须同步产出，不允许事后补做。
