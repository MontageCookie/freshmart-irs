# 业务边界

- 本工程为本科毕设《基于 Spring Boot + Vue 和 时间序列分析 的生鲜销售系统及智能补货系统》项目开发

1. 多角色协同  
   - 必须包含角色：店长 / 库管 / 收银 / 系统管理员 / 顾客  
   - 角色分工形成业务闭环，而非仅权限区分  

2. 进销存完整链路  
   - 入库 → 销售 → 扣减 → 预测 → 补货 → 再入库  

3. 顾客购买完整链路
   - 顾客注册 → 登录 → 浏览商品 → 加入购物车 → 下单 → 支付 → 订单确认  

4. 智能补货闭环  
   - 基于时间序列分析调用本地Python科学计算库（scikit-learn）进行预测。
   - 具体公式：$$Q_{suggest} = Q_{predict} \times T_{cycle} + S_{safety} - I_{current}$$ （其中 $Q$ 为数量，$T$ 为周期，$S$ 为安全库存，$I$ 为当前库存），自动生成建议采购单。

5. 预警机制  
   - 库存安全阈值预警与商品过期预警

6. 前端UI与数据可视化展示
