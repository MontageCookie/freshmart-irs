# 开发约束规则

API规范：严格遵循 RESTful 设计，资源使用复数名词，响应体格式统一。接口文档须通过 Knife4j 生成并维护。
注释要求：类、方法及关键字段均需添加功能、参数、返回值等必要注释。
质量门槛：后端核心逻辑单元测试覆盖率达90%以上，前端需使用 TypeScript 并定义类型。任务完成前必须通过严格自测。
日志与异常：统一采用 debug 级别记录日志。后端统一使用 SLF4J + Logback 作为日志实现。
提交与审计：所有代码变更须经用户审计，禁止在开发过程中自主提交 Git 记录。
库表规范：库表设计不允许使用外键。
