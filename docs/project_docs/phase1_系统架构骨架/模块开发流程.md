# 闭环模块的标准执行模板


### Step-0：【驱动层】实施方案 (Implementation Plan)

**目标**：确认执行路径，冻结技术方案。

* **任务拆解**：针对本模块，明确 Step-1 到 Step-4 的原子级任务清单。
* **技术选型**：明确本模块涉及的关键算法（如线性回归）、核心组件或第三方库。
* **重难点预警**：指出可能存在的逻辑返工点（如库存并发、效期计算），并给出处理策略。
* **验收前置**：明确该模块的“最短业务闭环”是什么，定义可演示的最小单元。

---

### Step-1：【契约层】设计规格 (Design Spec)

**目标**：定义开发约束，作为编码的唯一真理来源。

* **数据契约**：DDL 变更清单。冻结字段名、类型、索引、**字符串枚举值**。
* **API 契约**：冻结路径、Method、请求/响应 Body、**统一错误码**。
* **逻辑规则**：关键业务逻辑伪代码（如： 计算公式）、状态机流转图。

---

### Step-2：【实现层】后端开发与单测

**产物**：符合契约的代码 + 单元测试。
**标准**：Service 层逻辑 100% 覆盖；**Knife4j 接口通过 Mock 数据可跑通**。

---

### Step-3：【表现层】前端开发

**产物**：Vue 组件 + TS 类型定义。
**标准**：页面可呈现核心业务流；**异常状态（401, 403, 500）有统一反馈**。

---

### Step-4：【闭环层】集成联调

**标准**：**端到端（E2E）闭环**；数据流从页面操作准确反馈至数据库快照。

---

### Step-5：【交付层】论文素材收口

**产物**：最终 ER 图、核心接口截图、算法效果图、论文章节（实现部分）简述。
