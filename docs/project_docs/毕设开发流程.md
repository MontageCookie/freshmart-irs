## 3. 单一闭环模块的标准执行模板（复用）

### Step-0 模块定义文档
- 模块目标
- 参与角色
- 业务流程图
- 输入 / 输出
- 异常分支
- 数据落库点

【验收】
- 流程能完整走通
- 每一步都有可观测结果

---

### Step-1 数据契约冻结
- 涉及表 / 字段 / 索引清单
- 本模块新增或扩展的 DDL

【验收】
- DDL 可执行
- 不破坏既有核心结构

---

### Step-2 API 契约冻结
- 接口列表
- 请求 / 响应示例
- 权限与错误码

【验收】
- Knife4j 可调接口
- 示例完整

---

### Step-3 后端实现 + 单测
- Controller / Service / Mapper
- 核心业务逻辑
- 日志与异常

【验收】
- 单测覆盖达标
- 接口可调通

---

### Step-4 前端实现
- 页面 / 路由 / 组件
- TS 类型定义
- 接口调用

【验收】
- 页面可走最短业务路径
- 错误可见

---

### Step-5 联调验收
- 按角色模拟完整流程
- 校验数据库变化

【验收】
- 闭环完整
- 数据正确
- 可截图

---

### Step-6 阶段收口（论文素材入口）
- 架构图 / ER 图 / 流程图
- 接口截图 / 功能截图
- 本模块对应论文小节素材点

【验收】
- 素材可直接写进论文

---

## 使用方式总结

- 用户负责：
  - 决定下一个 Step
  - 确认验收清单
  - 最终通过与否
- Trae 负责：
  - 按文档与验收清单生成代码
- 本模板目标：
  - 每一步都能跑
  - 每一步都能验
  - 每一步都能变成论文素材
